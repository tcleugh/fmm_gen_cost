# Contract: UBODT (Upper Bounded Origin-Destination Table)

A precomputed shortest-path table consumed by **FMM only** (`src/mm/fmm/`). Other matchers do online Dijkstra and don't read this file.

Generated by the `ubodt_gen` executable (`src/app/ubodt_gen_app.cpp`) and consumed by `fmm` via `UBODT::read_ubodt_file(filename, multiplier)`.

## Accepted formats

`UBODT::read_ubodt_file` dispatches by extension:

| Extension | Reader |
|---|---|
| `.bin` | `read_ubodt_binary` — Boost `binary_iarchive` deserialization |
| `.csv` / `.txt` | `read_ubodt_csv` — `sscanf` of `;`-delimited rows |

Any other extension throws `std::runtime_error("File format not supported: <path>")`.

## CSV record layout

Semicolon-delimited (`;`). One header row followed by data rows. Each data row is parsed via:

```c
sscanf(line, "%d;%d;%d;%d;%d;%lf",
       &r->source, &r->target,
       &r->first_n, &r->prev_n, &r->next_e,
       &r->cost);
```

| Column | Type | Field | Meaning |
|---|---|---|---|
| 1 | int | `source` | Source node ID |
| 2 | int | `target` | Target node ID |
| 3 | int | `first_n` | First node on the shortest path from source (after source) |
| 4 | int | `prev_n` | Penultimate node on the shortest path (before target) |
| 5 | int | `next_e` | First edge on the shortest path |
| 6 | double | `cost` | Total path cost from source to target |

Example:

```text
source;target;first_n;prev_n;next_e;cost
1;2;1;1;5;0.142
1;3;1;2;5;0.250
2;1;2;2;7;0.180
```

The header line itself is skipped via a single `fgets`; column names are ignored.

## Binary record layout

Same six fields in the same order, serialized via Boost.Serialization's `binary_iarchive`. The exact byte layout is whatever Boost emits for `int` / `double` on the producing platform — **not portable across architectures**.

If you generate a binary UBODT on one machine and run FMM on another with a different endianness or `int` size, the load will produce garbage. Use CSV for cross-machine workflows.

## `multiplier`

`read_ubodt_file` takes a `multiplier` argument the caller derives from the network (`multiplier = network.get_node_count()`). This is used to hash `(source, target)` pairs into a bucket index inside the in-memory hash table. The file does not store this value — it must match between generation and consumption, otherwise lookups silently miss.

## Field-name overrides

None. The UBODT layout is positional, not header-driven.

## Validation

- File-existence is checked by the CLI config (`FMMAppConfig`).
- Row count is *estimated* (not exact) before allocation to size the hash table.
- Malformed CSV rows produce undefined values from `sscanf` — there is no per-row validation.

Generate via `ubodt_gen` to avoid hand-writing this file.
